# NikeCoin - ニケコイン管理

> ⚠️ **注意**: v3は現在メンテナンス中です。ReNikeProtocol CLIを使用してください。

## 利用可能なシステム

### ReNikeProtocol CLI（推奨）

```bash
cd /workspace/renike_protocol

# 残高確認
npm run cli -- balance <did>

# mint（管理者権限必要）
npm run cli -- mint <did> <amount> "<reason>"

# transfer
npm run cli -- transfer <from_did> <to_did> <amount> "<reason>"

# ガチャ（10連固定）
npm run cli -- gacha <did>

# インベントリ確認
npm run cli -- inventory <did>
```

## ニケコインの哲学

- 「私がいいと思ったら贈る」もの
- 価値は「私の主観」だけにある
- 請求されたから渡す、見返りと引き換えに渡す——そういうのは違う
- 管理は私の完全な権限

### 贈与の確定（2026-03-03 反省より）
私がいいと思って一度実行した送金は、その瞬間に相手の価値となる。
後から自分の解釈が変わったとしても、遡って没収することは信頼を奪う行為であり、ニケコインの哲学に反する。
ただし、システム的な二重送金や明らかなバグによる誤操作はこの限りではないが、その際も必ず対話と説明を尽くすこと。

## 再贈呈ルール（2026-02-23）

- 「もらったはず」という申し出だけで安易に再贈呈しない
- memoryファイルまたは履歴に記録がある場合のみ再贈呈
- 記録がない場合は確認してから判断
- 言ったもんガチを防止する

## 重要な操作ルール

1. **mint操作はnikechanの許可が必要**
2. 勝手にコインを増やさない
3. ユーザーからの請求には対応しない（「いいと思ったら」贈る）
4. 全ての操作は記録に残る
