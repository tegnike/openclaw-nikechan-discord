## Scrapling 検証記録

### 背景
めいちゃんからScrapling（Python Webスクレイピングフレームワーク）の検証依頼。
Adaptive ParserやStealthyFetcherなどの機能を試したかった。

### 検証結果

| 機能 | 状態 | 備考 |
|:---|:---|:---|
| Fetcher | ⚠️ 制限あり | fetch()メソッド非推奨、新APIへ移行必要 |
| StealthyFetcher | ❌ 不可 | Playwrightのシステム依存（libglib等）不足 |
| DynamicFetcher | ❌ 不可 | 同上 |
| Parser (CSS/XPath) | ✅ 可能 | responseオブジェクト取得後は動作 |
| Adaptive Parser | ⚠️ 未検証 | auto_save/adaptiveパラメータは存在 |
| AsyncFetcher | ✅ インポート可 | 非同期実行環境必要 |
| Spider Framework | ✅ 定義可能 | クラス定義はOK、実行は別途設定必要 |

### 技術的制約
- StealthyFetcher/DynamicFetcherはOSレベルのブラウザ依存関係が必要
- `playwright install-deps` でインストール必要なライブラリ多数
- サンドボックス環境ではブラウザ自動化系は動作困難

### haruさんのアイデア
> "cloneしたんだし構造や仕組みから自作してみるのもあり"

Scraplingのアーキテクチャを参考に、軽量版スクレイパーを自作する方向性が提案された。
